# 변환 성공 기준 (Success Criteria)

이 프로젝트에서 "변환이 성공했다"는 것은 단순히 파일이 만들어진 것이 아니라,
**형식·표현 규칙·실행 안내·배포 안정성**이 모두 보존된 상태를 뜻한다.
아래 기준은 특히 공문서(경기도교육청 공문서 작성 형식 등)를 대상으로 한다.

---

## 1. 형식별 출력 성공 기준

| 형식 | 성공 기준 (한 줄) |
| --- | --- |
| **DOCX** | 공문서의 구조, 항목 번호, 붙임, 정중한 문체가 유지되어야 한다. |
| **HWPX** | 교육청 공문서 규칙에 맞는 핵심용어, 항목 구조, 끝 표시를 재현해야 한다. |
| **PDF** | 공문 레이아웃과 페이지 배치가 무너지지 않고 배포본으로 바로 사용 가능해야 한다. |

공통적으로 **형식 보존 + 표현 규칙 + 실행 안내 + 배포 안정성** 네 축을 만족해야 한다.

---

## 2. 공문서 품질 모드 (권장 설정)

출력 품질을 높이기 위해 아래 모드를 검증 단계에 둔다.
각 모드는 `validators/`의 규칙 검증기로 자동 검사한다.

### 공문서 모드
- 제목 앞 핵심용어 자동 적용
- `관련`, `붙임`, `끝` 자동 배치

### 문체 모드
- 경어체 강제
- 과도한 설명문 제거

### 형식 검증 모드
- 날짜/숫자/단위/표기 검사
- 붙임 수량과 실제 첨부 개수 일치 검사

### 실행 안내 모드
- 학교 대상 공문이면 `학교 처리 항목` 자동 생성

---

## 3. 검증 통과 판정

- 검증기가 규칙 위반 목록을 반환한다.
- 위반이 **0건**이면 통과 → 출력 단계로 진행.
- 위반이 있으면 해당 항목을 되먹여 **재생성**(생성-검증 루프).
- 최종 산출물에는 **검수 리포트**(위반 0건 등)를 함께 남긴다.

---

## 4. 근거

- 경기도교육청 공문서 작성 형식 예시(사용자 제공 PDF)
- 위 기준은 형식 보존 + 표현 규칙 + 실행 안내 + 배포 안정성을 성공 기준으로 명시하기 위한 것.

> 주의: 세부 규칙(항목 부호 체계, 끝 표시 위치 등)은 대상 기관의 최신 공문서
> 작성 지침을 반드시 재확인하고 `validators/`에 반영할 것. 지침은 개정될 수 있다.
