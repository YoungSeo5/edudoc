# docs/AGENTS.md

`docs/` contains durable project documentation.

## Responsibility

Use this folder for long-lived project documentation.

## Rules

- Docs must reflect actual implementation status.
- Do not claim planned features are implemented.
- Do not use docs as a dumping ground for generated outputs.
- Do not duplicate long policies across many files.
- Permanent architecture documentation belongs here, not in `tasks/`.

Use honest terms:

- planned
- fallback
- experimental
- partially stabilized
- smoke-tested
- not layout-perfect
- project-local
